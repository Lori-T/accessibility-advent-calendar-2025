Touch targets need to be large enough for users to tap accurately on mobile and tablet devices. Small targets make interfaces difficult or impossible to use.

What you're checking today

You're checking that interactive elements like buttons and links provide a comfortable minimum tap area.

Quick 5-minute check

  • open your page on a mobile device or use Chrome DevTools mobile view
  • inspect a button or link using Inspect
  • hover over the element to view its dimensions
  • touch targets should be at least 44px by 44px
  • check spacing between targets so users don't tap the wrong element

The WCAG 2.2 skill behind this

2.5.5 Target Size (Enhanced) - interactive elements must be large enough to activate easily.

Why this matters

Larger touch targets support users with motor impairments, people using one hand, and anyone using a small device.

Try this bonus check

Try tapping your buttons while wearing gloves. Can you still hit the right target?
