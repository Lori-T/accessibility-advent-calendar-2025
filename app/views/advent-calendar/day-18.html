Interactive elements need meaningful labels that match what users see on screen. This is especially important for speech recognition and screen reader users.

What you're checking today

You're checking that the visible label is included in the control's accessible name.

Quick 5-minute check

  • inspect a button or input using Inspect
  • open the Accessibility panel in devtools
  • check the Accessible Name shown there
  • confirm it matches the visible label (for example, a button reading “Search” must expose “Search”)
  • avoid using aria-label if visible text is already present

The WCAG 2.2 skill behind this

2.5.3 Label in Name - the accessible name should include the visible label text.

Why this matters

Mismatched labels confuse users who rely on speech commands or assistive technologies. Consistency makes interactions predictable and reliable.

Try this bonus check

Turn on a screen reader and navigate to the control. Does it announce what you expect?
