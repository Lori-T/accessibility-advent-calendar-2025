The elves have created a “workshop status board” to show all the tasks they are working on. In a hurry, they picked heading styles that looked good rather than following a logical order.

Your mission today

Check that headings follow a clear, logical structure and reflect the hierarchy of the content.

Quick 5-minute check

  • identify the main page heading - there should usually be only one
  • check that subheadings follow a logical order (for example, h2 then h3)
  • make sure headings are not skipped just to achieve a visual style
  • avoid using headings for text that is not really a section heading

The WCAG 2.2 skill behind this

1.3.1 info and relationships - information, structure and relationships must be conveyed programmatically, not just visually.

Why this matters

Many screen reader users navigate by headings. If levels are inconsistent or used purely for appearance, it becomes hard to understand the structure of the page.

Bonus challenge

Use a browser extension or dev tools to view a list of headings on a real page. Does the order make sense without seeing the visual layout?
