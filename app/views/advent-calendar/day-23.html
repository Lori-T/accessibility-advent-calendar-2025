Some interactions are triggered by device motion, such as shaking or tilting. Users must be able to disable these features and use an alternative input method.

What you're checking today

You're checking that any motion-based interactions have simple alternatives and can be turned off.

Quick 5-minute check

  • look for features activated by shaking, tilting or device movement
  • check that these interactions are not required to complete tasks
  • ensure there is a button or control that performs the same action
  • confirm that motion inputs can be disabled in settings

The WCAG 2.2 skill behind this

2.5.4 Motion Actuation - provide alternatives to motion-based interactions.

Why this matters

Motion actuation can be difficult or impossible for users with motor impairments or people using fixed devices. Alternatives increase reliability and safety.

Try this bonus check

Try limiting motion on your device. Does the interaction still work using an alternative?
