New elves have to register for their shifts using an online form. The form was put together quickly using placeholder text instead of proper labels.

Your mission today

Check that every input, select and textarea has an explicit, associated label.

Quick 5-minute check

  • pick a form page in your service
  • check that each field has a visible label
  • inspect the HTML - labels should be linked to inputs using for and id
  • make sure placeholder text is not being used as the only label

Test the elf registration form

Below is a simplified version of the form. One of the fields relies only on a placeholder and has no proper label.

{# ❌ Deliberate issue: no label, only placeholder #}

Try navigating this form with a screen reader. “Elf ID code” will not be announced as a label, making the field harder to understand.

The WCAG 2.2 skill behind this

1.3.1 info and relationships and 3.3.2 labels or instructions - form controls must be properly labelled and instructions must be clear.

Why this matters

Without labels, form fields can be confusing or unusable, especially for screen reader users, people with cognitive impairments or people using autofill tools.

Bonus challenge

Inspect a real form in your service. Can you find any fields that only use placeholder text instead of labels?
