Form fields rely on clear, connected labels. Without proper labels, screen readers cannot announce what a field is for, and some users will be unable to complete the form.

What you're checking today

You're checking that every form field has a properly associated label that describes its purpose.

Quick 5-minute check

  • right-click a form field and select Inspect
  • look for a <label> element linked to the field using for and id
  • confirm that each input has a unique id
  • check that labels are visible and not replaced by placeholder text
  • for radios or checkboxes, look for a fieldset and legend describing the group

The WCAG 2.2 skill behind this

4.1.2 Name, Role, Value - elements must have the correct labels, roles and properties.

Why this matters

Some users cannot complete forms without properly linked labels. Labels also benefit speech recognition users and people with cognitive impairments.

Try this bonus check

Click the label text. Does it place the cursor into the input? If not, the label is not connected properly.
