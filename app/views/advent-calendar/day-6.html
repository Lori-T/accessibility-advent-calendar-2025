Day 6: Check focus visibility in the Reindeer Health Monitor

The Reindeer Health Monitor is used by Santa HQ to keep track of each reindeer’s wellbeing during the busiest time of year. Some elves have reported losing their place when navigating the system using only a keyboard.

Today, you will check whether it is always clear which element is currently selected when moving through the page using the keyboard.

Your mission today

Check that all interactive elements show a clear and visible focus indicator.

Quick 5-minute check

  • use the Tab key to move through the page
  • check that focus is always visible
  • focus should be easy to see and not subtle
  • focus must not rely on colour alone

Test the Reindeer Health Monitor

The section below contains one deliberate focus visibility issue.

✔ View Blitzen’s health record

Try using the keyboard only. Is it always clear where focus is?

What this checks for

2.4.7 focus visible - users must be able to see which element currently has keyboard focus.

Why this matters

Keyboard users rely on visible focus to understand where they are on a page. If focus is missing or unclear, users can become disoriented or unable to complete tasks.

How to check this

Focus visibility issues are best found using manual testing. Automated tools such as the Workshop Accessibility Verification Engine (WAVE) are not reliable for checking focus visibility.

  • use Tab and Shift + Tab to move through the page
  • check that focus is always clearly visible
  • pay close attention to custom styled buttons and controls
  • find out more about keyboard accessibility in last year’s advent calendar, Day 11

Bonus challenge

Zoom the page to 200 percent and then 400 percent and repeat the keyboard test. Is the focus indicator still easy to see?
