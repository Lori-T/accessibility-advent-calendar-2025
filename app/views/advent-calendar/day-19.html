Error messages help users understand what went wrong and how to fix it. They must be clear, specific and linked to the relevant field.

What you're checking today

You're checking whether your form provides clear error messages that help users recover from mistakes.

Quick 5-minute check

  • submit a form with an empty required field
  • look for an error summary at the top of the page
  • check for inline error messages next to each field
  • ensure errors state what went wrong and how to fix it
  • inspect the field to confirm it is linked to the error message using aria-describedby

The WCAG 2.2 skill behind this

3.3.1 Error Identification - errors must be identified and described clearly.

Why this matters

Clear error messages reduce frustration and help users complete tasks successfully, especially people with cognitive or memory impairments.

Try this bonus check

Turn on a screen reader and submit the form again. Can you easily find and correct the errors?
