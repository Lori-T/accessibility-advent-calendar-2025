Day 11: Check table structure in the Reindeer Flight Schedule

The Reindeer Flight Operations Team publishes a daily schedule showing which reindeer are taking off, when, and with which pilot elf. But in today’s update, one of the elves formatted a header using visual styling only — so it looks wrong and is not announced correctly by screen readers.

Your mission is to check whether each column header has been marked up semantically.

Your mission today

Find the column that visually looks like a header but isn’t one.

Quick 5-minute check

  • column headers must use <th>
  • headers must have scope="col" or row
  • visual styling alone does not make something a header
  • screen readers rely on semantic structure, not appearance

Review the flight schedule

One of the column “headers” is actually marked up as a normal cell. It even looks slightly wrong.

Reindeer Take-off time ❌ Pilot elf
(styled like a header, but not a real header)
Dasher 18:00 Fig
Dancer 18:10 Mistle

What this checks for

1.3.1 Info and relationships – structure must be conveyed in markup, not just visually.

Why this matters

People using screen readers rely on proper table headers to understand how data relates to the rest of the table. When a header is only styled visually, the structure becomes unclear.

Bonus challenge

Turn on a screen reader. Which column name is not announced before each cell?
