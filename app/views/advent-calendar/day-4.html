Some users increase text size to read more comfortably. Your content must remain usable when text is scaled significantly.

What you’re checking today

You’re checking that resizing text does not cause content to overlap, disappear, or become difficult to read.

Quick 5-minute check

  • increase text size or zoom using your browser’s settings (Chrome, Firefox, Edge → Zoom → 200%)
  • check headings, menus, buttons and forms for overlap or hidden text
  • ensure text wraps naturally and remains fully readable
  • check both browser zoom and system-level text resizing options

The WCAG 2.2 skill behind this

1.4.4 Resize Text - users must be able to increase text size up to 200% without losing content or functionality.

Why this matters

Many people with low vision or reading difficulties rely on text resizing. If your layout breaks, it becomes inaccessible.

Try this bonus check

Change only the text size (not the page zoom). Does your layout still work?
