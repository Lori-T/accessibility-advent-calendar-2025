Keyboard accessibility is essential for people who cannot use a mouse or trackpad. If a page cannot be operated with a keyboard alone, it is not accessible.

What you’re checking today

You’re checking that all interactive elements - links, buttons, menus, forms and controls - can be reached and used using only the keyboard.

Quick 5-minute check

  • click near the top of your page to set a starting point
  • press Tab to move forward through interactive elements
  • press Shift + Tab to move backwards
  • check that the focus order follows the visual layout of the page
  • ensure no elements are skipped or unreachable
  • activate buttons and links using Enter or Space

The WCAG 2.2 skill behind this

2.1.1 Keyboard - everything must be usable without a mouse.

Why this matters

Many users rely entirely on the keyboard. Strong keyboard support also improves compatibility with screen readers, voice control software and assistive devices.

Try this bonus check

Open a menu or modal with the keyboard. Can you tab into it, move around, and close it again?
