Day 7: Check form labels on the Elf Task Submission Form

Elves submit daily tasks through a central DEA-managed form. However, some input fields are missing labels, which means screen reader users cannot understand what information is expected.

Today you’ll review a simplified version of the form and spot the missing label.

Your mission today

Check that every form field has a clear, programmatic label.

Quick 5-minute check

  • every input must have a descriptive label
  • placeholder text must not be used instead of labels
  • associate labels using for and id
  • screen readers should announce the label naturally

Test the Task Submission Form

One field below has a deliberate accessibility issue.

What this checks for

4.1.2 name, role, value - form fields must expose correct labels to assistive technologies.

Why this matters

Without labels, users who rely on screen readers cannot complete forms accurately or confidently.

Bonus challenge

Turn on a screen reader and tab through the form. Which field is confusing?
