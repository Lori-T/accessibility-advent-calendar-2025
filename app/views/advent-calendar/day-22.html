Some features rely on gestures like swiping, dragging or pinching. These interactions must have simple alternatives for users who cannot perform complex gestures.

What you're checking today

You're checking that users can complete actions without needing multi-point or path-based gestures.

Quick 5-minute check

  • look for interactions that require dragging, swiping or pinching
  • check for a simple alternative like a button or link
  • test whether the interaction works with the keyboard
  • ensure users cannot trigger actions unintentionally

The WCAG 2.2 skill behind this

2.5.1 Pointer Gestures - avoid requiring complex gestures and provide alternatives.

Why this matters

Complex gestures can exclude users with motor impairments or those using assistive technologies. Simple alternatives improve usability and consistency.

Try this bonus check

Try completing the gesture using only the keyboard. If you cannot, there should be another way.
