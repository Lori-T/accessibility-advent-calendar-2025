Day 12: Check ARIA use on the Sleigh Launch Dashboard

The Sleigh Launch Dashboard brings together all the checks needed before Santa takes off - fuel status, wind speed, reindeer readiness and more. But a developer elf has added lots of unnecessary ARIA roles, and even broken one of the widgets by giving it the wrong role entirely.

Today you’ll review a small slice of the interface and look for incorrect or over-engineered ARIA.

Your mission today

Identify incorrect or unnecessary ARIA attributes.

Quick 5-minute check

  • do not add ARIA where native HTML is enough
  • ARIA roles must describe the element’s purpose
  • incorrect ARIA can break accessibility
  • use ARIA only when absolutely necessary

Test the Sleigh Launch Dashboard

One widget has a deliberate ARIA issue.

❌ This is a div with role="button" but no keyboard support.

What this checks for

4.1.2 name, role, value - ARIA must correctly expose the purpose of controls.

Why this matters

Misusing ARIA can make controls seem interactive when they are not, confusing users and assistive technologies.

Bonus challenge

Try navigating with a screen reader. Which element pretends to be a button but doesn’t behave like one?
