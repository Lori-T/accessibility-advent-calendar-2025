The elves have built a “present packing dashboard” to track gift wrapping progress. It looks festive, but some of the buttons lose their outline when you tab to them.

Your mission today

Check that it is always clear which element currently has keyboard focus.

Quick 5-minute check

  • use Tab and Shift + Tab to move around the page
  • at each step, check you can clearly see where focus is
  • make sure focus is not hidden behind colour alone (shape, border or outline all help)
  • ensure focus is visible on links, buttons, form controls and custom components

Test the present packing dashboard

Below is a row of controls used by the elves. One of the buttons has a deliberate accessibility issue: the focus outline has been removed for aesthetic reasons.

{# ❌ Deliberate issue: inline style removes the focus outline #}

Try tabbing through the buttons. Can you see focus on all three, or does one disappear?

The WCAG 2.2 skill behind this

2.4.7 focus visible - keyboard focus must always be visible.

Why this matters

Users who navigate with the keyboard need a clear visual indicator of where they are. If focus is not visible, it is easy to activate the wrong control or lose track of progress on a page.

Bonus challenge

Use your browser’s dev tools to temporarily remove the inline styles from the “Mark as urgent” button. How does the focus indicator change?
